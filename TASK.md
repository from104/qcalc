## 📅 최근 작업 히스토리

- **계산 기록 관리 로직 분리** `완료`

  - `useRecordManager` 컴포저블 신설하여 레코드 관련 기능 집중화
  - 파일 임포트/익스포트, 삭제 등 모든 레코드 관련 로직을 한 곳에서 관리
  - 의존성 주입 패턴 적용으로 컴포넌트 간 인스턴스 공유 문제 해결

  - `useMainLayout` 컴포저블에서 레코드 관련 로직 제거
  - `NarrowLayout`과 `WideLayout`이 `useRecordManager` 인스턴스를 직접 관리
  - 관심사의 분리로 코드 유지보수성 향상

- **설정 내보내기/가져오기 리팩토링** `완료`
  - 보안 강화: 환율 API 키 등 민감한 정보 제외
  - 최적화: 불필요한 런타임 상태 및 캐시 데이터 제외
  - 파일 크기 감소 및 개인정보 보호 개선

#### ⌨️ 단축키 개선

- **추가**: 계산 기록 폰트 크기 조절 (`Ctrl+[`, `Ctrl+]`)
- **제거**: 전체 기록 삭제 단축키 (`Ctrl+d`) - 실수 방지

#### 🐛 버그 수정

- 계산 기록 파일 임포트 버튼이 동작하지 않던 문제 해결
- `useSettingsManager.ts` 등 여러 파일의 ESLint 및 TypeScript 오류 수정

## 📝 계획된 작업

### 🔴 우선순위: 높음

#### 1. 결과 음성 출력 (TTS)

- **상태**: `계획됨`
- **설명**: 계산 결과를 음성으로 읽어주는 접근성 기능
- **구현 방안**:
  - Web Speech API (`speechSynthesis`) 연동
  - 설정 페이지에 TTS 활성화/비활성화 옵션 추가
  - 음성 속도, 음높이 등 세부 설정 제공
  - 다국어 지원 (한국어, 영어 등)
- **예상 작업 기간**: 2-3일
- **관련 이슈**: 접근성 향상

#### 2. Firebase 테마 공유 커뮤니티

- **상태**: `계획됨`
- **설명**: 사용자가 만든 테마를 다른 사용자와 공유할 수 있는 커뮤니티 기능
- **구현 방안**:
  - Firebase Firestore를 사용한 테마 저장소 구축
  - 테마 업로드/다운로드 UI 구현
  - 인기 테마 랭킹, 검색 기능
  - 사용자 인증 (Firebase Auth)
- **예상 작업 기간**: 1-2주
- **선행 요구사항**: 사용자 테마 설정 기능 (완료됨)

### 🟡 우선순위: 중간

#### 3. 모바일 앱 구현 (Capacitor: iOS)

- **상태**: `계획됨`
- **설명**: iOS 플랫폼용 네이티브 앱 빌드 및 배포
- **구현 방안**:
  - Apple 개발자 계정 설정 및 인증서 관리
  - Xcode 프로젝트 설정 및 빌드 환경 구축
  - iOS 네이티브 기능 처리 (알림, 파일 접근 등)
  - App Store 제출 및 배포
- **예상 작업 기간**: 2-3주
- **비고**: Android 버전은 이미 구현 완료

#### 4. 계산기 음성 조작 (STT)

- **상태**: `계획됨`
- **설명**: 음성 명령으로 계산기를 조작할 수 있는 접근성 기능
- **구현 방안**:
  - Web Speech API (`SpeechRecognition`) 연동
  - 자연어 처리 로직 구현 ("삼 더하기 오는?", "123 곱하기 45" 등)
  - 음성 인식 피드백 UI
  - 다국어 지원
- **예상 작업 기간**: 1-2주
- **관련 이슈**: 접근성 향상, TTS와 함께 사용 가능

### 🟢 우선순위: 낮음

#### 5. 스킨 기능

- **상태**: `아이디어`
- **설명**: UI 레이아웃을 통째로 변경하는 기능
- **구현 방안**:
  - 스킨 아키텍처 설계 (컴포넌트 교체 시스템)
  - 기본 스킨, 미니멀 스킨, 레트로 스킨 등 제공
  - 스킨 관리 UI 구현
- **예상 작업 기간**: 3-4주
- **비고**: 구조적 변경이 크므로 신중한 설계 필요

#### 6. 광고 기능 (수익화)

- **상태**: `보류`
- **설명**: 배너 광고를 통한 앱 수익화
- **구현 방안**:
  - AdMob SDK 연동
  - 광고 표시 위치 결정 (UX 고려)
  - 프리미엄 버전 (광고 제거) 검토
- **비고**: 사용자 경험을 해치지 않는 선에서 신중하게 접근

  - 설정 파일 내보내기/가져오기

### UI/UX

- [x] **반응형 레이아웃**
  - 좁은 화면용 `NarrowLayout`
  - 넓은 화면용 `WideLayout` (2단 구조)
  - 자동 전환
- [x] **키보드 단축키 지원**
  - 탭 전환 (Ctrl+1~4, Ctrl+Tab)

- [x] **사용 팁 시스템**
  - 앱 시작 시 팁 표시
  - 도움말 페이지

  - `useMainLayout`: 레이아웃 로직 관리

  - `useSettingsManager`: 설정 관리
  - `useKeyBinding`: 키보드 단축키 관리
- [x] **상태 관리 (Pinia)**
  - calcStore, uiStore, settingsStore, themesStore
- [x] **플랫폼 지원**
  - Electron (Windows, Linux 데스크톱)
  - Capacitor (Android 모바일)
  - Web (브라우저)

## 📖 작성 가이드

### 작업 상태 레이블

- `계획됨`: 구체적인 계획이 수립된 작업
- `진행 중`: 현재 작업이 진행되고 있는 항목
- `완료`: 작업이 완료되고 테스트까지 마친 항목
- `보류`: 일시적으로 중단된 작업
- `아이디어`: 구체화되지 않은 아이디어 단계

### 작업 우선순위

- 🔴 **높음**: 핵심 기능, 버그 수정, 사용자 경험 개선
- 🟡 **중간**: 부가 기능, 플랫폼 확장
- 🟢 **낮음**: 실험적 기능, 장기 프로젝트

### 히스토리 작성 규칙

- 최신 날짜가 위에 오도록 작성
- 각 작업은 카테고리별로 분류 (리팩토링, 버그 수정, 기능 추가 등)
- 관련 파일이나 커밋 해시가 있으면 명시
- 작업의 목적과 결과를 간결하게 설명
